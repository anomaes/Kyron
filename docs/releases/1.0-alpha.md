# Kyron 1.0-alpha

Kyron 1.0-alpha is the first public prerelease of the trusted-internal workflow
orchestration engine. It is intended for evaluation and controlled deployments where
workflow authors, repositories, and operators are trusted.

## Included in this release

- Exact-commit workflow and transitive sub-workflow snapshots.
- Bash, repository Script, and Pi Prompt process nodes with checkpointed waves.
- Human-feedback, sub-workflow, and bounded review-loop control nodes.
- GitLab and GitHub OAuth, change-request, approval, and webhook integration.
- Project roles, approval policies, immutable gate eligibility, and audit records.
- A visual workflow builder with typed interfaces, reusable node templates, and
  project-local definition reviews.
- Encrypted user credentials, redacted process output, durable attempts, cancellation,
  and checkpoint-based recovery.
- Searchable operational, workflow-authoring, API, and deployment documentation.

## Operating boundaries

This alpha retains Kyron's documented version-one deployment model:

- Run exactly one backend worker.
- Allow only trusted users to author workflows or register repositories.
- Treat Bash and Script nodes as direct code execution. Prompt processes have a
  worktree write boundary, but remain trusted because they retain read, credential,
  network, and compute access.
- Use durable storage for PostgreSQL, repository clones, worktrees, and run output.
- Complete the environment-dependent acceptance checks for each enabled code host before
  promoting an installation to production use.

See the [acceptance record](/acceptance), [security model](/deployment/security), and
[production deployment guide](/deployment/) for the corresponding verification and
operational requirements.

## Version identifiers

The product release and Git tag are `1.0-alpha` and `v1.0-alpha`. Package manifests use
the equivalent native prerelease forms: Python `1.0a0` and npm `1.0.0-alpha`.
